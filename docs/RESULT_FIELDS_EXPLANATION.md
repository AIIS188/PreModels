## 返回结果结构

```python
{
    "x_today": {...},      # 今日发货计划（吨）
    "trucks": {...},       # 建议车数
    "mixing": {...},       # 混装明细
    "arrival_plan": {...}  # 到货诊断
}
```

---

## 1. x_today - 今日发货计划

### 内部格式（模型输出）
```python
Dict[Tuple[str, str, str, str], float]
# 键：(warehouse, cid, category, date)
# 值：发货吨数
```

**示例**:
```python
{
    ("W2", "HT-001", "A", "2026-03-15"): 49.13,
    ("W2", "HT-002", "A", "2026-03-15"): 49.57,
    ("W1", "HT-003", "A", "2026-03-15"): 47.65,
}
```

### JSON 序列化格式（返回结果）
```python
{x_today_key: tons}
# 键："{warehouse}_{cid}_{category}_{date}"
# 值：发货吨数（float）
```

**示例**:
```json
{
    "W2_HT-001_A_2026-03-15": 49.13,
    "W2_HT-002_A_2026-03-15": 49.57,
    "W1_HT-003_A_2026-03-15": 47.65
}
```

### 字段含义

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `warehouse` | str | 发货仓库 | `"W1"`, `"W2"` |
| `cid` | str | 合同编号 | `"HT-001"` |
| `category` | str | 品类名称 | `"A"` (动力煤), `"B"` (焦煤) |
| `date` | str | 发货日期 | `"2026-03-15"` |
| `tons` | float | 发货吨数 | `49.13` |

### 业务含义

- **决策变量**: LP 模型的核心输出，表示今日从某仓库发往某合同某品类的吨数
- **产能约束**: 同一仓库同一品类的所有合同发货量之和不超过该仓库该品类的日产能
- **合同完成**: 所有仓库发往某合同的总量用于完成该合同的剩余需求

## 2. trucks - 建议车数

### 内部格式（模型输出）
```python
Dict[Tuple[str, str, str], int]
# 键：(warehouse, cid, date)
# 值：建议车数（整数）
```

**示例**:
```python
{
    ("W2", "HT-001", "2026-03-15"): 2,
    ("W2", "HT-002", "2026-03-15"): 2,
    ("W1", "HT-003", "2026-03-15"): 2,
}
```

### JSON 序列化格式（返回结果）
```python
{trucks_key: count}
# 键："{warehouse}_{cid}_{date}"
# 值：建议车数（int）
```

**示例**:
```json
{
    "W2_HT-001_2026-03-15": 2,
    "W2_HT-002_2026-03-15": 2,
    "W1_HT-003_2026-03-15": 2
}
```

### 字段含义

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `warehouse` | str | 发货仓库 | `"W1"`, `"W2"` |
| `cid` | str | 合同编号 | `"HT-001"` |
| `date` | str | 发货日期 | `"2026-03-15"` |
| `count` | int | 建议车数（向上取整） | `2` |

### 计算逻辑

```python
# 1. 按 (warehouse, cid, date) 聚合吨数
total_tons = sum(x_today[(w, cid, k, t)] for all k)

# 2. 获取估重画像 (mu, hi)
mu, _ = weight_profile.get((w, receiver, k), (35.0, 37.0))

# 3. 计算车数（向上取整）
trucks = ceil(total_tons / mu)
```

**示例**:
```
W2_HT-001: 49.13 吨 ÷ 35 吨/车 = 1.4 → 2 车
```

### 业务含义

- **混装支持**: 同一 (warehouse, cid, date) 的不同品类可以拼车
- **估重画像**: 基于历史磅单数据的平均每车吨数（默认 35 吨）
- **向上取整**: 确保所有货物都能装车，避免超载

## 3. mixing - 混装明细

### 内部格式（模型输出）
```python
Dict[Tuple[str, str, str], Dict[str, float]]
# 键：(warehouse, cid, date)
# 值：{category: tons} 品类吨数分配
```

**示例**:
```python
{
    ("W2", "HT-001", "2026-03-15"): {"A": 49.13},
    ("W2", "HT-002", "2026-03-15"): {"A": 49.57},
    ("W1", "HT-003", "2026-03-15"): {"A": 47.65},
}
```

### JSON 序列化格式（返回结果）
```python
{mixing_key: {category: tons}}
# 键："{warehouse}_{cid}_{date}"
# 值：{品类名称：吨数}
```

**示例**:
```json
{
    "W2_HT-001_2026-03-15": {"A": 49.13},
    "W2_HT-002_2026-03-15": {"A": 49.57},
    "W1_HT-003_2026-03-15": {"A": 47.65}
}
```

### 字段含义

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `warehouse` | str | 发货仓库 | `"W1"`, `"W2"` |
| `cid` | str | 合同编号 | `"HT-001"` |
| `date` | str | 发货日期 | `"2026-03-15"` |
| `category` | str | 品类名称 | `"A"`, `"B"` |
| `tons` | float | 该品类的吨数 | `49.13` |

### 业务含义

- **混装指导**: 告诉装车人员每车应该装哪些品类、各多少吨
- **多品类合同**: 一个合同可能包含多个品类（如动力煤 + 焦煤）
- **配载优化**: 简化版本，实际可能需要更复杂的配载算法

**多品类示例**:
```json
{
    "W1_HT-001_2026-03-15": {
        "动力煤": 30.0,
        "焦煤": 20.0
    }
}
```
表示这车应该装 30 吨动力煤 + 20 吨焦煤，共 50 吨。

## 4. arrival_plan - 到货诊断

### 内部格式（模型输出）
```python
Dict[Tuple[str, str], float]
# 键：(cid, arrival_date)
# 值：期望总到货吨数（在途 + 新增）
```

**示例**:
```python
{
    ("HT-001", "2026-03-16"): 85.5,
    ("HT-001", "2026-03-17"): 3.2,
    ("HT-003", "2026-03-16"): 50.0,
}
```

### JSON 序列化格式（返回结果）
```python
{arrival_plan_key: tons}
# 键："{cid}_{arrival_date}"
# 值：期望总到货吨数
```

**示例**:
```json
{
    "HT-001_2026-03-16": 85.5,
    "HT-001_2026-03-17": 3.2,
    "HT-003_2026-03-16": 50.0
}
```

### 字段含义

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `cid` | str | 合同编号 | `"HT-001"` |
| `arrival_date` | str | 预计到货日期 | `"2026-03-16"` |
| `tons` | float | 期望总到货吨数 | `85.5` |

### 计算逻辑

```python
# 总到货 = 在途预测 + 新增发货预测
arrival_plan[(cid, d)] = pred_mu[(cid, d)] + A_new[(cid, d)]

# pred_mu: 在途报单按延迟分布预测的到货量
# A_new: 今日发货按延迟分布预测的到货量
```

**延迟分布示例**:
```python
delay_dist = {0: 0.03, 1: 0.95, 2: 0.02}
# 3% 当日到，95% 隔日到，2% 2 日到
```

**计算示例**:
```
今日发货 100 吨
→ 今日到货：100 * 0.03 = 3 吨
→ 明日到货：100 * 0.95 = 95 吨
→ 后日到货：100 * 0.02 = 2 吨
```

### 业务含义

- **到货诊断**: 预测未来每天的到货量，帮助仓库安排接货
- **均衡性检查**: 对比 `q_star`（均衡目标），检查是否集中到货
- **在途跟踪**: 包含已在途的报单预测，不仅仅是新发货

```

└─────────────────────┬───────────────────────────────────────┘
                      │
        ┌─────────────┼─────────────┐
        │             │             │

        ▼

```

## 使用示例

### Python 代码解析

```python
result = optimizer.run(today_date="2026-03-15", H=10)

# 1. 今日发货计划
for key, tons in result["x_today"].items():
    w, cid, k, t = key.split("_")
    print(f"{t} 从 {w} 发往 {cid} ({k}): {tons:.2f} 吨")

# 2. 车数建议
for key, trucks in result["trucks"].items():
    w, cid, t = key.split("_")
    print(f"{t} {w}→{cid}: {trucks} 车")

# 3. 混装明细
for key, details in result["mixing"].items():
    w, cid, t = key.split("_")
    for category, tons in details.items():
        print(f"{t} {w}→{cid} {category}: {tons:.2f} 吨")

# 4. 到货诊断
for key, tons in result["arrival_plan"].items():
    cid, date = key.split("_", 1)
    print(f"{date} {cid} 预计到货：{tons:.2f} 吨")
```

## 关键参数

### 模型参数（影响输出）

| 参数 | 默认值 | 说明 | 影响 |
|------|--------|------|------|
| `alpha_short` | 3000.0 | 缺口惩罚权重 | 越大越优先完成合同 |
| `beta_balance` | 2.0 | 均衡偏差权重 | 越大到货越均衡 |
| `gamma_waste` | 80.0 | 过期浪费权重 | 越大越避免过期发货 |
| `H` | 10 | 规划窗口（天） | 影响长期规划 |
| `stability_weight` | 0.1 | 稳定性权重 | 越大计划抖动越小 |

### 配置参数（影响计算）

| 参数 | 默认值 | 说明 | 影响 |
|------|--------|------|------|
| `rho_intransit` | 0.9 | 在途置信系数 | 影响在途预测 |
| `max_daily_ratio` | 1.5 | 每日发货上限 | 防止集中发货 |
| `weight_profile` | (35, 37) | 估重画像 | 影响车数计算 |
| `delay_profile` | {0:0.03, 1:0.95} | 延迟分布 | 影响到货预测 |
