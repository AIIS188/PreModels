- 按 lane(warehouse, cid, day) 聚合吨数后换算车数
- 新增 `get_mixing_details()` 返回品类分配明细

**代码**：
```python
# common_utils_v2.py
def suggest_trucks_from_tons_plan(..., allow_mixing=True):
    # 按 (w, cid, t) 聚合吨数
    total_tons = sum(tons for k, tons in category_tons)
    trucks = ceil(total_tons / weighted_mu)
```

**效果**：
- 减少总车数约 15-20%

### 3. 在途期望修复

**问题**：
- 在途期望计算包含了今天到达的量
- 导致今日发货目标偏低

**解决**：
- 在途期望只计算 today+1 及以后到达的
- 今天到达的不影响今日发货（因为今天快结束了）

**代码**：
```python
# simple_system_v2.py
intransit_mu_valid = intransit_total_expected_in_valid_window(
    c.cid, pred_mu, today + 1, c.end_day  # 从明天开始算
)
```

**效果**：
- 今日发货目标更合理
- C1 发货从 85.4 吨提升到 99.1 吨

### 5. 到货上限约束优化

**问题**：
- 原约束限制"在途 + 新增"的总到货
- C2 day10 在途 138 吨 >> max_daily 50 吨，导致无法新增发货

**根因分析**：
```
C2 需求分析:
  Q=900, delivered=520, future_intransit=13.7
  remaining=367.7, T=11, q_star=33.4/天
  max_daily=50.1, day10_intransit=138.0
  day10 可新增 = max(0, 50.1-138.0) = 0 吨 ❌
```

**解决**：
- 改为限制每日发货量（x[w,cid,k,t] 的和）
- 在途量不影响新增发货约束

**代码**：
```python
# complex_system_v2.py
for t in range(today, min(today + H, c.end_day + 1)):
    ship_expr = pulp.lpSum(x[(w, cid, k, t)] for ...)
    model += (ship_expr <= max_daily_ship, f"MaxDailyShip_{cid}_{t}")
```

**效果**：
| 指标 | 修复前 | 修复后 | 改进 |
|------|-------|-------|------|
| C1 发货 | 0 吨 | 138.8 吨 | ✅ |
| C2 发货 | 0 吨 | 44.0 吨 | ✅ |
| C1 到货 Std | 149.9 | 14.5 | -90% |
| C2 到货 Std | 25.2 | 16.1 | -36% |
